cant_datos = np.linspace(2, 39, 20).astype(int)
errores_train_prom = []
errores_val_prom = []
errores_train_std = []
errores_val_std = []

train_index = list(X_train.index)

for i in cant_datos:
    errores_train = np.zeros(100)
    errores_val = np.zeros(100)
    
    for k in range(100):
        # Selecci√≥n aleatoria de √≠ndices
        train_index_i = random.sample(range(len(X_train)), i)
        
        # Crear subconjuntos de entrenamiento y validaci√≥n
        X_train_i = X_train.iloc[train_index_i]
        y_train_i = y_train.iloc[train_index_i]

        model = SGDRegressor(max_iter=1000000, eta0=1e-4, random_state=semilla, penalty= None)
        model.fit(X_train_i, y_train_i)

        # Predecir en el subconjunto de entrenamiento y validaci√≥n
        y_train_pred = model.predict(X_train_i)
        y_val_pred = model.predict(X_val)
        
        # Calcular el error cuadr√°tico medio
        errores_train[k] = mean_squared_error(y_train_i, y_train_pred)
        errores_val[k] = mean_squared_error(y_val, y_val_pred)
    
    # Promedio y desviaci√≥n est√°ndar de los errores
    errores_train_prom.append(np.mean(errores_train))
    errores_val_prom.append(np.mean(errores_val))
    errores_train_std.append(np.std(errores_train))
    errores_val_std.append(np.std(errores_val))

# Agregar errores base
errores_train_prom.append(MSE_train)
errores_val_prom.append(MSE_val)
errores_train_std.append(np.std([MSE_train]))  # Desviaci√≥n est√°ndar para un solo valor es 0
errores_val_std.append(np.std([MSE_val]))      # Desviaci√≥n est√°ndar para un solo valor es 0

# Crear un DataFrame con los resultados
resultados = pd.DataFrame({
    'Subconjunto': cant_datos.tolist() + ['L√≠nea base'],
    'Error_Entrenamiento': errores_train_prom,
    'Error_Validacion': errores_val_prom,
    'STD_Entrenamiento': errores_train_std,
    'STD_Validacion': errores_val_std
})

print(resultados)

Sesgo y varianza¬∂

En la gr√°fica de error contra tama√±o de subconjunto se puede observar que cuando el tama√±o de subconjunto es de 2, el sesgo es demasiado grande puesto que el modelo se ajusta solamente a esos dos puntos, dando como resultado un modelo que tiene mucho error en los datos de validaci√≥n. El resto de los modelos tienen menos sesgo y por lo tanto menos error, por debajo de 1100, sin embargo, hay mucha varianza entre ellos ya que se tiene un rango de error 200 hasta 1050, adem√°s de que la desviaci√≥n estandar de los errores para cada tama√±o de subconjunto de entrenamiento tambi√©n es considerable, esto significa que el cambio en la selecci√≥n aleatoria de datos influye significativamente en el desempe√±o (varianza alta). Esto quiere decir que la complejidad o par√°metros del modelo deber√≠an cambiarse.

Ajustes obtenidos¬∂

En el caso del modelo que utiliza 2 elementos para entrenar se tiene un modelo con underfit, puesto que el error es demasiado grande y no logra hacer predicciones adecuadamente. Por otro lado, cuando tenemos muchos elementos es probable que el modelo no generalice bien y no obtenga buenas soluciones para datos no vistos, esto quiere decir que el modelo sufre de overfit. A lo largo de todos los modelos se tiene un error relativamente grande, esto puede ser debido a que los par√°metros seleccionados para entrenar los modelos no son los mejores.

Con estas condiciones, el modelo que tiene menor error y menor variaci√≥n es el que utiliza 5 muestras para entrenamiento.
